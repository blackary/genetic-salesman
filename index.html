Traveling Salesman Problem.

If you have a number of different cities to visit, spread all over the map, how do you find the shortest path from one, to the next, to the next, and finally back to the first city?

A simple problem with 4 cities, but what about 10? 100? If there are N cities, there are N! different paths. If N is 30, N! is 30x29x28...x3x2x1 = 2.65x10^32

Two Possible Solutions.

  • Greedy Algorithm
  • Genetic Algorithm

Greedy Algorithm

Basic idea: Start with 1 city, find the next closest city, and repeat until you've run out of cities left (then go back to the first)

Advantages Relatively fast algorithm. O(N^2), rather than N! - a significant improvement. Gives a decent solution.

Disadvantages Never gives an ideal solution. Typically the first "end" of the path is inefficient. Doesn't consider the "big picture".
